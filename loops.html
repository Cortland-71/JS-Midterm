while loop

Okay so in this section we're going to take a look at two kinds of loops in Java. The while loop and the for loop. Let's first take a look at the while loop.


The while loop is the simplest form of a loop in Java. Much like the if statement in the previous section, it only executes if a condition is true. The big difference from the if statement however is that it will continue to execute over and over again until the condition is false. Let's look at an example.

SMALL MOVES

Loops are one of the most valuable tools at your disposal. They are efficient for iterating over lists and performing tasks in succession. Learn to use loops and you will cut out a lot of tedium. More advanced techniques might include nested loops. We won't be going into this here but know that you can nest loops inside of other loops. This creates a two dimensional effect and can be very powerful.

for loop

The for loop can look a little complicated at first so let's demystify them a bit in this section. First, the for loop is a loop like the while loop however unlike the while loop, you must know how many times the iteration will happen. Let's start with what's happening at line 5. This is the begining of the for loop and as you can see the first section is declairing an int called "i" and assigning the value to 0. This will be the counter. Then there is a semi colon ; as to mark the end of the statment.


Next, the loop will test this counter against some condition as you can see in the second section. In this case it's asking if our counter "i" is infact less than 10, if it is then jump down inside the block and print "fizz".


Once the code inside the for loop block is done executing it will jump back to the top of the loop and will hit the third section.


The ++ is an increment operator. It will add one to a varaible. In this case it will add one to our counter "i" making it 1 after the first itteration. Once it's done incementing it will then check the condition again in the second section. If our counter is still less then 10, do the code block again, an d so on.
